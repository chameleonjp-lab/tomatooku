  情報が無いため未確認**(下記参照)。

## 7. レビュー指摘への対応(2026-06-30)

| # | 指摘 | 対応 |
| --- | --- | --- |
| A1 | スコアにサーバー上限がない | クライアント `ranking.js` で 0〜180000 に clamp。`SUPABASE_SETUP.md` の SQL も `least(180000, greatest(0, …))` に修正。再計算する強化版 RPC も追記 |
| A2 | クリア演出 850ms がスコアに混入 | `GameSession` を実プレイ時間積み上げ式に変更。`clearCurrentStage()` でクリア瞬間に確定し、演出/待ち時間を除外。単体テストで検証 |
| A3 | tomatooku と tomatoku の混在 | `SUPABASE_SETUP.md` に登録 CSV 行を追記し `game_slug=tomatoku` / `game_url=tomatooku` を明示固定 |
| B5 | 初回スコアの表示 | ランキング表示に「初回 X」を副表示(`firstScore` がある場合) |
| B6 | 正解パターン重複 | 重複統計を本書・SPEC に明記。`pickStages` を正解パターン非重複選出に変更(2000 プレイで 3 種相異なりを検証) |
| C7 | E2E の Chromium 固定パス | `scripts/launch.js` を追加し、固定パスが無ければ既定の `chromium.launch()` にフォールバック |
| C8 | 色だけのエリア区別 | 盤面エリア境界に太枠(`.edge-*`)を描画し、色覚に依存せず区別可能に |

### 正解パターン重複の統計(参考)
30 問は **10 種類の正解パターン**を共有(盤面=エリア形は全て相異なる)。最大グループは
4 問共有(例: T001/T004/T005/T006、T012/T015/T020/T025 など)。`pickStages` の
非重複選出により、同一プレイ内では必ず異なる正解パターンになる。
