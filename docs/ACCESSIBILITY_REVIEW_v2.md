# トマトオク v2 画面・アクセシビリティ確認

## 1. 対象

v2実装計画の第5回「画面とアクセシビリティ」に対応する。

対象:

- ホームの保存・送信説明
- ゲーム中の誤タップ理由
- モーダルフォーカス
- キーボード操作
- 読み上げ
- 動きを減らす設定
- 小型画面
- 外部リンク
- チュートリアル文言
- ランキング公開後のE2E期待値

ゲームルール、補正タイム式、公式問題、ランキングRPC、Supabaseデータは変更しない。

## 2. 監査で見つかった問題

### 2-1. 保存説明が不正確

画面下部に「保存されるのはプレイヤー名だけ」と表示されていた。

実際には、公式プレイ時に次が公開ランキングへ保存される。

- プレイヤー名
- 補正タイム
- プレイ回数

`localStorage`へ保存する値と、Supabaseへ送信する値を分けて説明する必要があった。

### 2-2. E2Eが旧送信ゲートを期待

公式送信を有効化した後も、E2Eは「公開準備中」という表示を期待していた。

送信RPCをスタブし、登録成功とベスト・初回表示を検証する内容へ変更した。

### 2-3. モーダルのフォーカス管理不足

モーダルを開いてもフォーカスが起点に残り、`Tab`キーで背景へ移動できた。閉じた後のフォーカス復元もなかった。

### 2-4. 誤タップ理由が視覚・読み上げで不明

赤い揺れだけでは、行、列、エリア、隣接のどのルールに違反したか分からなかった。

### 2-5. 動きを減らす設定が未対応

盤面、モーダル、チュートリアルのアニメーションが`prefers-reduced-motion`を考慮していなかった。

### 2-6. 320px幅のHUDが窮屈

4項目を1行へ配置していたため、小型端末ではラベルと数値の余白が不足した。

### 2-7. チュートリアル文言が旧スコア方式

「高スコア」という文言が残っており、補正タイムの小さい順という現行仕様と一致していなかった。

## 3. 実装

### 3-1. 独立モジュール

追加:

```text
src/accessibility.js
src/accessibility.css
```

ゲームロジックへアクセシビリティ処理を混在させない。

### 3-2. プレイヤー名説明

名前欄へ`aria-describedby="name-privacy"`を設定し、以下を表示する。

- 公式プレイの公開保存内容
- 練習は送信しない
- 本名・メール・電話番号を入力しない

### 3-3. 誤タップ理由

盤面クリック後にDOM状態から次を判定して表示する。

- 同じ行
- 同じ列
- 同じエリア
- 上下左右・斜めの隣接
- 5個上限

`#game-status`は`role="status"`と`aria-live="polite"`を持つ。

### 3-4. モーダル

- 開いたときに先頭の操作要素へフォーカス
- `Tab` / `Shift+Tab`を内部で循環
- 閉じたときに起点へ復元
- 背景スクロールを停止
- ダイアログ見出しを`aria-labelledby`で関連付け

### 3-5. 盤面・チュートリアル

- 盤面へ`role="grid"`と説明を追加
- チュートリアル進行を`role="progressbar"`として公開
- チュートリアル終了文を補正タイム方式へ更新

### 3-6. CSS

- `:focus-visible`
- 盤面セルの強調フォーカス
- `prefers-reduced-motion`
- `forced-colors`
- 320〜380pxでHUDを2列化
- 小型画面で外部リンクを縦積み
- 低い横画面の盤面・モーダル調整
