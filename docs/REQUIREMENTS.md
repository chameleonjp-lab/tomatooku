# トマトク 要件書 (REQUIREMENTS)

## 1. ゲーム概要

『トマトク』は、5×5 の畑に 🍅 を配置する短時間ブラウザパズル。
1プレイでランダムに選ばれた **3ステージ** を連続で解き、クリアまでの速さと
正確さをスコアで競う。ランキングは「最高スコア・高い順」。

- ジャンル: 論理パズル(いわゆる Queens / Star Battle の 5×5 エリア版)
- 1プレイの所要時間: 目安 1〜3 分
- 公開形態: 静的サイト(サーバー常駐処理なし)
- ランキング: Supabase(公開 anon key のみ)で他プレイヤーと比較

## 2. 対象端末

- **最優先: スマートフォンの縦持ちブラウザ**
- iPhone SE 相当(幅 320px)でレイアウトが破綻しないこと
- 横スクロールを発生させないこと
- ピンチズーム・ダブルタップズーム・長押し選択を可能な範囲で抑制
- 操作に遅延・引っかかりを出さない
- 配信容量は 3MB 以内(実測では数十 KB)
- 重い画像・音声・大規模ライブラリを使わない

## 4. パズルルール

盤面は 5×5。各ステージで 🍅 を 5 個置く。配置条件:

1. 各行に 🍅 は 1 個だけ
2. 各列に 🍅 は 1 個だけ
3. 各エリア(色分けされた畑の区画・5マス)に 🍅 は 1 個だけ
4. 🍅 同士は上下左右斜めで隣り合ってはいけない

25マスは 5 つのエリア(各 5 マス)に分割される。エリアは色で見分ける。

## 5. ステージ仕様

- 初期実装で **30ステージ** を用意(画像ではなく軽量データ)。
- データ形式(`src/stages.js`):
  ```js
  { id: "T001", difficulty: 1,
    regions: ["AABBC","ADBBC","ADDEC","AEEEC","DDCCC"],
    solution: [[0,0],[1,3],[2,1],[3,4],[4,2]] }
  ```
  - `regions`: 5 行の文字列。`A`〜`E` の 5 エリア。
  - `solution`: 正解の 🍅 配置 `[row, col]` × 5。
- 各ステージは生成時・出力後の二段階で検証(`scripts/verify_stages.js`):
  - 5×5 であること / エリアが 5 つ・各 5 マス
  - 解が存在し、**一意解**であること
  - `solution` がルールを満たし、ソルバ解と一致すること
- 難易度: ソルバ探索量(バックトラック量)の三分位で 1/2/3 に分類。各 10 問。
- **ランダム選出**: 難易度 1/2/3 の各グループから 1 問ずつ選び、3ステージを構成。
  異なる難易度から取るため同一プレイ内で重複しない。出題順は やさしい→ふつう→むずかしい。

## 6. 操作仕様

- マスをタップで 🍅 を置く。置いてあるマスを再タップで取り除く。
- **誤タップ**: 見えているルールに反する配置操作。
  - 同じ行 / 同じ列 / 同じエリアに既に 🍅 がある
  - 隣接(8方向)に既に 🍅 がある
  - 既に 5 個置いている
  - 誤タップ時は置かず、揺れ + 赤表示で通知。誤タップ数 +1、スコア減点。
  - **隠し答えで罰しない**: 最終解と違うマスでも、その時点でルール違反でなければ誤タップにしない。
- **ヒント**: 未配置の正解 🍅 を 1 つ確定配置。ヒント数 +1、大きく減点。
  - 進行不能を防ぐため、ヒント時に「正解でない配置」を除去して盤面を solution の部分集合へ正規化してから 1 マス追加する。
  - クリア済み・残り正解なしのときはボタンを無効化。

## 7. クリア判定

各ステージは次を満たした瞬間に自動クリア:

- 🍅 が 5 個 / 各行・各列・各エリアに 1 個ずつ / 隣接なし

クリアで短い演出を入れ次のステージへ。3ステージ目クリアで結果画面へ。

## 8. スコア仕様

ランキングは「スコアが高い順」。スコアはクリア速度を強く反映する。

```js
const BASE_SCORE = 180000;
const timePenalty = Math.floor(elapsedMs);     // プレイ全体の経過ミリ秒
const mistakePenalty = mistakeCount * 3000;
const hintPenalty = hintCount * 30000;
const score = Math.max(0, BASE_SCORE - timePenalty - mistakePenalty - hintPenalty);
```

- `elapsedMs` は **3ステージ通しの合計経過時間**(開始〜3ステージ目クリア)。
- 表示: スコア / クリアタイム / 誤タップ数 / ヒント回数 / ステージ数 / 送信状態。
- ランキング登録値は整数スコア。

## 9. ランキング仕様

| 項目 | 値 |
| --- | --- |
| game_slug | `tomatoku` |
| title | トマトク |
| top_ranking_type | best |
| score_order | desc |
| score_unit | pt |
| score_scale | 1 |
| score_decimals | 0 |
| score_label | スコア |
| first_score_label | 初回スコア |
| best_score_label | 最高スコア |

- Supabase 連携。**公開 anon key のみ**使用(secret key は使わない)。
- 送信はゲーム終了時に **1回だけ**(二重送信防止)。
- 送信中 / 成功 / 失敗の状態を結果画面に表示。通信失敗でも結果画面は表示する。
- 詳細・SQL は `docs/SUPABASE_SETUP.md`。

## 10. 保存仕様

- ブラウザに保存してよいのは **プレイヤー名のみ**(`localStorage["tomatoku.playerName"]`)。
- スコア / ランキング / ステージ進行 / クリア履歴は localStorage に保存しない。
- ランキングはサーバー側連携に委ねる。

## 12. 仮決め(不明点の扱い)

ランキング連携の既存実装(chameleonjp-lab 内の他ゲーム)は、本セッションの
GitHub アクセスが `chameleonjp-lab/tomatoku` のみに限定されており参照できなかった。
そのため依頼書に示された RPC インターフェース(`submit_score` / game_slug /
初回・最高スコア)に沿って、以下を **合理的な仮決め**として実装した。

- Supabase REST の RPC `submit_score(p_game_slug, p_player_name, p_score)` を呼ぶ。
- ランキング取得は RPC `get_ranking(p_game_slug, p_limit)`。
- RPC 名・引数名が既存ゲームと異なる場合は `src/ranking.js` の `CONFIG`
  (`submitRpc` / `rankingRpc` と各 `p_*` キー)だけ合わせれば差し替え可能。
- 接続情報(URL / anon key)は `index.html` の `window.TOMATOKU_CONFIG` で設定。
  未設定でもゲームは完全に動作し、ランキングのみ「未設定」表示になる。

詳細な根拠は `docs/IMPLEMENTATION_PLAN.md` のリスク節と `docs/SUPABASE_SETUP.md`。
