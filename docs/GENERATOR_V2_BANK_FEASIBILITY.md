# 生成器v2 84問候補バンク成立性監査

- 対象: `chameleonjp-lab/tomatooku`
- 対象工程: v2実装計画 第6回・Slice 3
- 監査種別: 全探索による成立性確認
- 監査結果: **BLOCKED BY CONSTRAINTS**
- 実行コマンド: `npm run audit:v2:bank`
- 固定結果: `generated/stage-bank-feasibility-v2.json`

## 1. 結論

現在の盤面契約をすべて維持し、回転・反転・エリア名変更を同一盤面として除外する場合、作成可能な一意解盤面は最大5種類です。

```text
目標canonical盤面数       84
正解配置                  14
連結領域分割              21,452
一意解付きラベル盤面      36
D4・ラベル正規化後         5
目標成立                  false
```

したがって「各エリア5マスを維持したまま、対称形を除外した84問バンクを作る」という計画は、探索回数を増やしても達成できません。

これは確率的生成器の性能不足ではなく、全候補を列挙した結果です。

## 2. 監査した契約

以下をすべて固定して全探索しました。

- 盤面は5×5
- エリアは5個
- 各エリアはちょうど5マス
- 各エリアは上下左右の4近傍で連結
- 各行に🍅は1個
- 各列に🍅は1個
- 各エリアに🍅は1個
- 🍅同士は上下左右・斜めで隣接しない
- 解は1個だけ
- 回転90・180・270度を同一視
- 左右・上下・主対角線・反対角線反転を同一視
- エリア名A〜Eの付け替えを同一視

## 3. 全探索方法

### 3-1. 正解配置

行ごとの🍅列を順列として列挙し、次を満たす14配置を取得します。

- 列重複なし
- 隣接する行の列差が2以上

### 3-2. 各エリア候補

各正解配置について、正解セルを各エリアの種とします。

各種セルについて次を満たす5マス集合をすべて列挙します。

- 対象種セルを含む
- 他エリアの種セルを含まない
- 合計5マス
- 上下左右で連結

### 3-3. 正確な領域分割

5エリアの候補集合から、重複せず25マスを完全に覆う組み合わせをすべて列挙します。

候補数の少ないエリアから選ぶことで探索を枝刈りしますが、候補自体は省略しません。

### 3-4. 一意解確認

完成した各領域分割を全探索ソルバへ渡し、解が1個だけであることを確認します。

### 3-5. canonical化

一意解盤面へD4の8変換を適用し、各変換後にエリア名を初出順でA〜Eへ正規化します。

辞書順最小の署名をcanonical署名とし、同一署名を1問として数えます。

## 5. 現行30問との関係

現行`src/stages.js`の30問も、同じD4・ラベル正規化を適用すると5種類へ集約されます。

canonical型ごとの現行問題数は次のとおりです。

```text
8問
7問
6問
6問
3問
```

現行30問が不正という意味ではありません。回転、反転、エリア配置方向の違いを別問題として扱っているため、プレイ上は30問として成立しています。

ただし、実装計画にある「対称形を除外しながら最低84問」とは両立しません。

## 8. 仕様変更候補

### A. 対称形を別問題として許容する

現行方式を拡張し、回転・反転を別問題として数えます。

利点:

- ゲームルールを変更しない
- 実装コストが低い

欠点:

- 見た目の向きが違うだけの問題が増える
- 「対称形除外」の要件を撤回する必要がある
- 84問でも実質的な構造は5種類

### B. 各エリア5マス固定を緩和する

5エリアと連結性は維持し、エリアサイズを可変にします。

例:

```text
各エリア3〜7マス
5エリア合計25マス
各エリアに🍅1個
```

利点:

- 行・列・エリア・隣接禁止というゲームの中心ルールを維持できる
- 盤面構造の組み合わせが大幅に増える可能性が高い
- 対称形除外を維持できる可能性がある

欠点:

- ステージ契約とvalidatorの変更が必要
- 小さいエリア・大きいエリアの視認性と難易度を再検証する必要がある
- 84問成立性を新契約で再監査する必要がある

### C. 盤面サイズまたは隣接ルールを変更する

例:

- 6×6へ拡張
- 斜め隣接禁止を緩和
- エリア数を変更

利点:

- 大幅な問題数増加が期待できる

欠点:

- ゲーム性、UI、タイマー、難易度が大きく変わる
- iPhone SE向け操作性を再設計する必要がある

## 9. 推奨

**候補B「各エリア5マス固定を緩和する」**を推奨します。

理由:

- 5×5盤面を維持できる
- 5個の🍅、各行・列・エリア1個、隣接禁止を維持できる
- スマートフォンUIを大きく変えずに済む
- 対称形除外と実質的な多様性を維持できる可能性が最も高い

ただし、新契約を採用する前に次の順で進めます。

1. エリアサイズ範囲の候補を比較
2. 各候補を全探索監査
3. 84canonical問以上が存在することを証明
4. 視認性・難易度への影響を確認
5. 契約を確定
6. 新バンクを生成
