# トマトオク v2 モード・補正タイム実装確認

## 対象

- 公式モードだけが送信対象となる契約

## 公式問題

公式問題は次の3問で実装上固定する。

```text
1. T001 / difficulty 1
2. T011 / difficulty 2
3. T021 / difficulty 3
```

起動時に次を検証する。

- 3件すべてが存在する
- IDが重複しない
- 難易度が1→2→3
- 正解配置署名が重複しない

不正な場合は公式プレイを開始しない。公開前に人間が3問を通しプレイし、難易度順と体感差を最終確認する。

## ランダム練習

難易度1・2・3の全組み合わせから、以下を満たす組だけを事前列挙する。

- IDが重複しない
- 正解配置署名が重複しない

有効な組が0件なら設定エラーとする。重複を許すフォールバックは使用しない。

## 補正タイム

ランキングへ渡す整数値は100分の1秒単位とする。

```text
floor(実時間ms / 10)
+ 誤タップ数 × 300
+ ヒント数 × 3000
```

表示時は`score / 100`秒、小数2桁とする。

例:

```text
実時間: 12.345秒
誤タップ: 2回
ヒント: 1回

12.34 + 6.00 + 30.00 = 48.34秒
ランキング値 = 4834
```

ミリ秒から100分の1秒への変換は切り捨てる。3分を超える値も上限で切らない。

## ランキング送信

- `official`: 送信対象
- `practice`: 常に送信しない
- play IDが空: 送信しない
- 同じplay ID: 1回だけ送信

ただし、現在は`public.games`への`tomatoku`登録と実疎通が未完了のため、`submissionsEnabled=false`で送信ゲートを閉じる。

ゲート解除条件:

1. `public.games`へ`tomatoku`を登録
2. `score_order=asc`
3. `score_unit=秒`
4. `score_scale=100`
5. `score_decimals=2`
6. テスト名で送信し、初回・ベスト・プレイ回数を確認
7. 実験場と詳細ランキングで表示を確認
8. テストデータを削除または明示管理
