## 1. v2の目的

現行の5×5論理パズルを維持しながら、公式ランキングの公平性、スコアの分かりやすさ、共通Supabase基盤との整合、スマートフォンでの安定性を改善する。

v2では「公式3問」と「ランダム練習」を分ける。ランキング対象は全員が同じ条件で遊ぶ公式3問だけとする。

## 2. 体験契約

### 2.1 この作品の特徴

Star Battle / Queens型の制約を、トマトと畑の区画として短時間で理解できる形にした3ステージパズル。見えているルール違反へ即時に反応し、隠し答えを理由にプレイヤーを罰しない。

### 2.2 変えてはいけない部分

- 各行にトマトは1個
- 各列にトマトは1個
- 各エリアにトマトは1個
- トマト同士は上下左右斜めで隣接しない
- 各盤面は一意解
- 誤操作は見えているルールだけで判定する
- タップで置き、再タップで外せる
- 公式ランキングでは全員の出題条件を揃える

### 2.3 対応方針

- 対応方針: 広域対応型
- 主保証端末: iPhone 17 Pro
- 検証端末: iPhone 11 Pro、iPad Pro 2018
- 主対象ブラウザ: Safari
- 補助自動検証: Chromium、可能ならWebKit

実機未確認の項目を「保証済み」と記載しない。

### 2.4 最も重い場面

25マス盤面、HUD更新、誤タップまたはクリア演出が同時に動くゲーム画面。ホーム・結果画面ではランキング通信が加わる。

### 2.5 反応速度

- 安定30fps以上を基本とする
- タップから視覚反応まで100ミリ秒以内を目標とする
- 意図しない100ミリ秒超の停止は公開停止条件とする

## 3. 対象端末と画面方向

- スマートフォン縦持ちを最優先
- iPhone 17 Pro、iPhone 11 Pro、iPhone SE相当幅で破綻しない
- iPad Proの縦・横画面でも操作不能にならない
- 横スクロールを発生させない
- セーフエリアへ主要ボタンを重ねない
- 文字サイズ変更時も主要情報を欠けさせない

全作品共通の容量上限は設けない。トマトオクは軽量構成を維持するが、3MB以内を製品要件にはしない。

## 4. 画面遷移

```text
home
  ├─ 公式3問 → countdown → playing → stageTransition → result
  └─ ランダム練習 → countdown → playing → stageTransition → result
```

結果画面からは、同じモードへの再挑戦、ホーム、詳細ランキング、実験場へ移動できる。

## 5. 公式3問モード

- ランキング対象
- 全プレイヤーが同じ3問を同じ順序で遊ぶ
- やさしい・ふつう・むずかしいを1問ずつ採用
- カウントダウン中は盤面を見せない
- 終了時にSupabaseへ1プレイ1回だけ送信する
- 同じランキング期間中は公式3問を変更しない
- 公式3問は`T001 / T011 / T021`をこの順番で固定する

## 6. ランダム練習モード

- ランキング対象外
- 承認済み84問完成バンクから難易度1・2・3をランダムに1問ずつ出す
- feature gate有効時のprimaryは`candidate-v2-variable-4-6-final`
- 読込・検証・時間切れ失敗時は既存30問の`legacy-v1`へfallbackする
- 一時fallback後は次の練習開始時に完成バンクを再取得する
- 結果画面へ「ランダム練習」と表示する
- Supabaseへスコアを送信しない
- 可変4〜6マス問題はこのモードだけへ先行導入する

日替わりランキングはv2初回公開に含めない。

## 7. パズルルール

5×5の盤面へトマトを5個置く。

1. 各行に1個
2. 各列に1個
3. 各エリアに1個
4. 上下左右斜めで隣接しない

条件を満たした瞬間に自動クリアする。各盤面は出荷前に一意解を検証する。

## 8. タイマー

- `performance.now()`を使う
- ホームから3・2・1・スタートを経て計測を開始する
- カウントダウン中は盤面を見せない
- プレイヤーが操作できる時間だけを計測する
- クリア演出とステージ間待機は含めない
- 次ステージは盤面描画完了後に計測を再開する
- バックグラウンド中も時間は経過扱いにする
- 必要に応じて復帰時に時間が進んだことを表示する
- ステージ別時間も保持する

## 9. 補正タイム

v2では、180,000点から減点する旧方式を廃止し、現在は次の補正タイム方式を使用する。

```text
補正タイム
= 実プレイ時間
+ 誤タップ数 × 3秒
+ ヒント数 × 30秒
```

Supabaseへ送る整数値:

```js
score =
  Math.floor(elapsedMs / 10)
  + mistakeCount * 300
  + hintCount * 3000;
```

- 100分の1秒単位
- 小さいほど良い
- 表示精度とランキング精度を一致させる

ランキングメタ情報:

```text
top_ranking_type: best
score_order: asc
score_unit: 秒
score_scale: 100
score_decimals: 2
score_label: 補正タイム
first_score_label: 初回タイム
best_score_label: ベストタイム
```

## 10. 誤タップとヒント

### 誤タップ

見えているルールに反する配置だけを誤タップとする。理由を短い文で表示する。

- 同じ横列にある
- 同じ縦列にある
- 同じエリアにある
- となりのマスには置けない
- すでに5個ある

### ヒント

ヒントは、正解ではない配置を取り除き、正しい位置を1つ置く。使用前に動作と30秒加算を明示する。

## 11. 結果画面

最低限、次を表示する。

- 補正タイム
- 実プレイ時間
- 誤タップによる加算
- ヒントによる加算
- ステージ別時間
- 初回記録
- ベスト記録
- 現在順位
- 公式または練習

サーバーから更新判定を得られない場合、「ベスト更新」と断定しない。

## 12. ランキング

実験場の共通Supabase契約へ合わせる。

送信RPC:

```text
submit_score
p_display_name
p_game_slug
p_score
p_client_version
```

取得RPC:

```text
get_best_score_ranking
get_first_try_ranking
```

要件:

- ブラウザではanon / Publishable keyだけを使う
- service_role keyを入れない
- 共有Supabaseの`submit_score`をゲーム専用SQLで置き換えない
- 公式モードだけを送信する
- 1プレイ1送信
- 通信失敗と0件を別状態で表示する
- `CLIENT_VERSION`でランキングに影響する変更を識別する

## 13. 保存・送信データ

端末内へ保存する情報:

```text
プレイヤー名のみ
```

サーバーへ送る情報:

```text
表示名
補正タイム
game_slug
client version
```

開始前に送信内容を説明する。本名、メールアドレス、電話番号を入力しないよう案内する。

表示名は送信前に次を行う。

- 前後空白を除く
- 連続空白を1つへまとめる
- 制御文字と見えない方向制御文字を除く
- 空文字を拒否する

## 14. 公開・実験場連携

- 標準公開先はCodeberg Pages
- GitHubは開発・レビュー用
- 現在の複数ファイル・ES Modules構成を維持できる
- 1ファイル化は任意の配布方法で、必須ではない
- ホームと結果に実験場への導線を表示する
- ホームと結果に詳細ランキングへの導線を表示する
- `public.games`へ`tomatoku`を登録し、共通ランキングを利用する
- URL群は公開先変更時に同時更新できるよう管理する
- Codeberg Pages反映後に公開URLと各導線を実機で確認する

## 15. 安定性と復帰

- クリア演出中の入力をロックする
- リタイア・再開始時に待機タイマーを解除する
- play/session IDで古い処理を無効化する
- 旧セッションが新セッションを進めない
- 通信失敗でもゲーム結果を失わない
- エラー時に再試行導線を表示する
