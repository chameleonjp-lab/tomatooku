# 生成器v2 可変エリアサイズ成立性監査

## 1. 目的

固定5マス契約では、D4変換とエリア名変更を同一視した一意解盤面が最大5問しか存在せず、84問目標は不成立だった。

本監査は、中心的なゲームルールを維持したまま、エリアサイズだけを最小限緩和した場合に84問以上が存在するかを、実装前に確認する。

## 2. 監査対象

```text
盤面                         5×5
エリア数                     5
エリアサイズ                 4〜6マス
合計                         25マス
各エリア                     上下左右の4近傍で連結
各行・各列・各エリア         🍅1個
🍅同士                       上下左右・斜めで隣接禁止
解                           1個だけ
重複除外                     D4変換＋エリア名初出順正規化
必要canonical盤面            84
```

`3〜7マス`まで広げる案も存在したが、まず中心仕様への影響が小さい`4〜6マス`だけを監査した。

## 3. 監査方式

監査は最大数を数える全探索ではなく、84個の相異なるcanonical盤面が実在することを証明するthreshold-witness方式である。

1. 5×5ルールを満たす正解配置14種類を順番に処理する
2. 各正解セルを含み、他の正解セルを含まない連結領域候補を列挙する
3. 各領域サイズを4・5・6マスに限定する
4. 5領域で25マスを完全被覆するexact-cover探索を行う
5. 可変サイズ対応ソルバで一意解を確認する
6. 解が探索対象の正解配置と一致することを確認する
7. D4変換とエリア名変更を正規化する
8. 新しいcanonical盤面を保存する
9. 84問へ到達した時点で停止する

したがって本監査は「最大数」を主張しない。84問以上が存在することだけを証明する。

## 4. 結果

```text
要求canonical盤面                    84
確認canonical盤面                    84
目標成立                              true
処理した正解配置                      1 / 14
対象正解配置                          0,2,4,1,3
訪問した連結完全分割                  9,524
一意解として確認した分割              84
監査方式                              threshold-witness
全候補の完全列挙                      false
```

最初の正解配置だけで84問へ到達したため、残り13配置を探索する必要はなかった。

## 5. エリアサイズ分布

84問のサイズプロファイルは次の2種類だった。

```text
4-4-5-6-6    65問
4-5-5-5-6    19問
```

5領域の合計はすべて25マスであり、3マス以下または7マス以上のエリアは必要なかった。

## 6. 固定成果物

```text
generated/stage-bank-variable-feasibility-v2.json
```

manifestには次を保存する。

- audit version
- 監査方式
- エリアサイズ範囲
- 必要問数と確認問数
- 訪問分割数
- 正解配置別候補数
- サイズプロファイル分布
- 84問の安定`stageId`
- canonical署名
- 盤面
- エリアサイズ
- 一意解

## 7. 再現コマンド

```bash
npm run audit:v2:variable-regions
npm run test:generator-v2:variable-regions
```

CIでは監査を再実行し、コミット済みmanifestとの差分が0であることを確認する。

## 8. バンク状態

```text

candidate-v2.status = blocked-by-constraints
candidate-v2-variable-4-6.status = feasible-pending-contract-approval
candidate-v2-variable-4-6.runtimeEnabled = false
candidate-v2-variable-4-6.rankingEligible = false
```

成立確認は仕様採用を意味しない。次を明示的に決定するまでゲームへ接続しない。

- エリアサイズ4〜6マスを正式仕様に採用するか
- 練習モードだけへ先行採用するか
- 84問をすべて候補バンクへ採用するか
- 人間向け難易度と近似盤面距離をどの基準で選別するか

## 10. 次工程
